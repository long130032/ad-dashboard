# 广告投放诊断看板

把创量后台导出的报表,变成一眼看清「**钱花得好不好、浪费在哪、谁家账户该管**」的看板。

不用装软件、不用连数据库。打开网页 → 上传后台导出的 CSV → 当场出图。每次上传覆盖上一批,只看当下这盘,不攒历史。

---

## 打开就能用

**网址:https://long130032.github.io/ad-dashboard/**

第一次打开是空的,三步出图:

1. 从创量后台导出报表 CSV(账户报表、项目报表都行,系统自己认)。
2. 进左侧「**数据导入**」,把导出的 CSV 一起选进去,点上传。
3. 完事。看板各页就有数据了。

> 上传的表会**记在你这台电脑的浏览器里**,关掉网页、重开、刷新都还在。换台电脑或换个浏览器,得重新传一次。

## 换一批新数据

还是在「数据导入」页:直接传新的 CSV 就会**覆盖**旧的;想先清空,点「清空当前数据」归零再传。

## 数据安全吗?

安全。这个网址虽然是公开的,但它**只是个空壳计算器**——你上传的报表**只在你自己浏览器里算**,一个字节都不会发到任何服务器。别人就算打开这个网址,看到的也是空白看板,没有你的数据。

(所以反过来:你也没法靠发网址给同事就让他们看到你的数;要给团队看,投屏或截图给他们。)

## 看板怎么读

| 页面 | 看什么 |
| --- | --- |
| **总览** | 整盘的钱花了多少、转化多少、有多少在浪费;哪几个项目最烧钱(集中度);最该管的问题账户排在最前。 |
| **项目** | 每个项目花了多少、效果如何,系统给的状态建议(主力 / 潜力 / 需关注 / 低消耗 / 问题)。 |
| **团队** | 每个优化师的盘子多大、带了多少问题账户。 |
| **媒体** | 钱在腾讯 / 巨量 / 磁力等各家怎么分的。 |
| **账户明细** | 逐个账户:属于哪一类问题、病灶在哪(没展示?没点击?没转化?)、给的处理建议。 |

### 三条要记住的看法

- **一切按"花的钱"排,不按账户个数。** 一堆没花钱的小号会把人带偏;真正该看的是钱去哪了。
- **CPA(转化成本)不能跨项目、跨人直接比。** 有的项目目标是"激活"(几毛一个),有的是"付费"(几十一个),放一起比没意义。所以团队页、媒体页的 CPA 只能参考。
- **真正的浪费是"成本偏高",不是 0 转化的小号。** 账户分五类,系统按「白花的钱」从多到少排。最肉疼的是那些**有转化、但在同项目里成本明显偏高**的账户(占大头);零转化的小号金额其实很小——**抓大放小**。

账户五类:零消耗(没花钱)/ 起量中(在跑还没出转化)/ 有消耗无产出(花了钱啥也没出,真问题)/ 成本偏高(出了转化但太贵)/ 正常。

## 维护说明(给改代码的人)

业务使用看到这里就够了,以下是给维护者的。

**更新线上看板**:改完代码,在项目目录里
```bash
git add -A && git commit -m "改了什么" && git push
```
推上去约 1 分钟自动重新部署(GitHub Actions),刷新网页生效。

**本地改 / 预览**:
```bash
npm install
npm run dev       # 本地开发,http://localhost:5173
npm run build     # 打包出 dist/
npm run preview   # 本地看打包后的成品
npm test          # 跑口径回归测试(需本地的 数据样本/)
```

**口径都在一处**:所有清洗、指标、诊断逻辑集中在 `src/compute/`,加新指标 / 新筛选只改这里,页面无需改。`src/lib/api.ts` 是页面取数的统一入口(浏览器内直接算,不走网络)。

**不在仓库里的东西**(本地保留,刻意不上公开仓库):真实数据 `数据样本/`、内部文档 `reference/`、依赖样本的口径测试。线上网站也从不包含任何真实数据。
